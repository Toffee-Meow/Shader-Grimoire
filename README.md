## ✨ 仓库简介
<blockquote>
  魔法书介绍
</blockquote>

&emsp;&emsp;个人🍭云端 Shader “代码”仓库，这里收录了我在游戏引擎里探索过的Shader、打磨过的材质、调试过的灯光等，同时也是我学习 **艺术向技术美术（TA）** 路上的成长印记。

&emsp;&emsp;渲染风格一般以 **三渲二/卡通渲染** 为主～嘿嘿，因为本人敲爱 **二次元卡通** 画风🍧，对 **角色卡渲** 尤为钟爱😻。

&emsp;&emsp;仓库上传的 **Shader** 将按制作内容（即 Shader 实现的具体视觉效果或功能目标）分类，内容贴合 **游戏引擎 / 技术美术（TA）** 领域的实际开发逻辑，计划涵盖：

- 角色渲染类 Shader
- 通用材质类 Shader
- 视觉特效类 Shader
- 后期处理类 Shader
- 风格化渲染类 Shader
- 性能优化类 Shader

&emsp;&emsp;本仓库除收录各类 Shader 外，还设有 **[引擎渲染合集](引擎渲染合集)** 板块，记录一些个人学习阶段的渲染练习作品，均为 **引擎实时渲染** 出图，涵盖地编场景、灯光布光、视觉特效、角色卡通渲染、三渲二效果等，欢迎查看🍬！

## 🔎 “食用”方法
<blockquote>
  魔法书使用方法
</blockquote>

&emsp;&emsp;<b>仓库内容介绍📌</b>
<p>

- 仓库里的 **[万象幻典.md](万象幻典.md)** 为<i>Shader分类清单</i>，记录仓库所有已上传的 **Shader/工程的名称、分类和内容** ，方便查找。
- 每个上传的Shader效果都会新建一个文件夹，文件名为对应Shader效果/内容，按照 **创作平台/软件 - 制作内容（Shader效果）** 分类，打开对应的code文件夹查阅即可。
- 每个Shader文件夹内都会有一个 **Readme.md** 文件，简述该文件/项目的内容和引擎实时渲染效果。
- 该项目仓库除Shader代码文件以外，可能还会存储如 **自定义渲染管线、工程资产文件、引擎脚本** 等等，可按照文件类型区分。
</p>

&emsp;&emsp;<b>如何使用？🚀</b>
<p>

- ​直接复用​​：代码/资产适配主流引擎（Unreal Engine、Unity，具体看文件），开箱即用或稍作修改；
- ​灵感参考​​：从简单的“UI渐变背景”到复杂的“角色卡渲”，看代码文件/工程文件，逐步拆解Shader逻辑，学习思路，再用到自己的Shader里；
- ​贡献创意​​：哪怕是一个小效果的改进、一点代码的优化、一行注释的优化，都欢迎你的指正和讨论！
</p>
